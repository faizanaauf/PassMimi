# Security Policy

Thank you for helping keep PassMimi secure.

Security is one of the primary goals of this project. If you discover a security vulnerability, please report it responsibly instead of disclosing it publicly.

---

# Supported Versions

The latest release is actively maintained and receives security updates.

| Version | Supported |
|---------|:---------:|
| Latest Release | ✅ Yes |
| Older Releases | ❌ No |

Please update to the latest version before reporting a security issue.

---

# Reporting a Security Vulnerability

If you discover a vulnerability, **do not create a public GitHub Issue.**

Instead, report it privately by email.

📧 **Security Contact**

**Email:** fai_ebook@proton.me

Please include the following information whenever possible:

- Description of the vulnerability
- Steps to reproduce
- Expected behavior
- Actual behavior
- Screenshots (if applicable)
- Operating system
- Python version (if running from source)
- Proof of Concept (PoC), if available

Providing detailed information helps us investigate and resolve the issue more quickly.

---

# What Happens After You Report

After receiving your report, we will:

1. Confirm receipt of your report.
2. Investigate the issue.
3. Validate the vulnerability.
4. Develop and test a fix.
5. Release a security update if necessary.
6. Credit the reporter where appropriate (unless anonymous disclosure is requested).

We appreciate responsible disclosure and will work to resolve confirmed vulnerabilities as quickly as possible.

---

# Scope

Examples of security issues include:

- Authentication bypass
- Remote code execution
- Arbitrary file access
- Privilege escalation
- Dependency vulnerabilities
- Sensitive information disclosure
- Unsafe password handling
- Security flaws affecting password analysis

General bugs, feature requests, or UI issues should be reported through GitHub Issues instead.

---

# Security Principles

PassMimi is designed around the following principles:

- Privacy-first password analysis
- Local processing whenever possible
- No password transmission to external servers
- No telemetry or user tracking
- Transparent open-source development
- Secure coding practices

These principles guide both current development and future features.

---

# Responsible Disclosure

Please allow reasonable time for the vulnerability to be investigated and fixed before publicly disclosing any details.

Responsible disclosure helps protect users while a fix is being prepared.

---

# Third-Party Dependencies

PassMimi relies on several open-source libraries and security datasets.

If a vulnerability affects one of these dependencies, we recommend updating to the latest version as soon as a patched release becomes available.

---

# Future Security Enhancements

The long-term security roadmap includes:

- Secure encrypted password vault
- Enterprise password policy auditing
- Website password validation API
- Security rule customization
- Automated dependency monitoring
- Continuous security testing
- Optional AI-assisted password recommendations
